# D-Index

Десктопний застосунок для роботи з власним рушієм даних, що зберігає записи в оперативній пам'яті. Дозволяє додавати, шукати, редагувати та видаляти записи, імпортувати їх із CSV або генерувати тестові набори, зберігати поточний стан у файл-знімок (`.didx`) і відновлювати його, переглядати журнал операцій з можливістю Undo/Redo, а також запускати вбудований аналіз продуктивності.

---

## Можливості

- Додавання, оновлення, видалення та пошук записів за ідентифікатором, діапазоном Id або префіксом ключа
- Імпорт даних з CSV-файлу та генерація до 1 000 000 тестових записів
- Збереження повного стану у бінарний знімок `.didx` та відновлення з нього з перевіркою цілісності
- Транзакційний журнал останніх 100 операцій з Undo / Redo
- Перегляд метрик і графіка активності на дашборді
- Запуск аналізу продуктивності з HTML-звітом
- Пагінація таблиць по 50 записів, фільтрація журналу за типом операції
- Робота повністю офлайн, без мережевих залежностей

## Інструкція користувача

Бічна панель ліворуч містить навігацію між сторінками. Усі дії виконуються миттєво й локально; тривалі операції показують індикатор завантаження.

### Дашборд

Огляд стану сховища.

- **Всього записів** — загальна кількість записів у сховищі (включно з видаленими).
- **Активних** — записи, доступні для пошуку.
- **Видалених** — логічно видалені записи (місце ще зайняте, доки не очистити сховище).
- **Розмір індексу (МБ)** — приблизне споживання пам'яті записами.
- **Останній пошук (мс)** — час виконання останнього пошуку.
- **Глибина індексу** — показник «здоров'я» внутрішнього дерева; якщо стає завеликим, з'являється попередження.
- **Сторінок пам'яті / Заповненість / Операцій у журналі** — рядок з допоміжною статистикою.
- **Швидкі дії** — кнопки переходу до додавання, пошуку та збереження знімка.
- **Активність за часом** — графік розподілу записів по інтервалах часу їх створення.
- **Кнопка «Оновити метрики»** — примусове оновлення всіх показників.

### Пошук

Пошук записів у трьох режимах.

- **За ідентифікатором** — точний пошук за числовим Id; повертає один запис.
- **За діапазоном** — повертає всі записи, чий Id лежить у діапазоні `Від … До`.
- **За ключем (префікс)** — повертає всі записи, ключ яких починається з введеного тексту (нечутливо до регістру).
- **Таблиця результатів** — Id, ключ, обрізані дані, час створення, статус.
- **Кнопка «✕»** у рядку таблиці — видаляє відповідний запис.
- **Лічильник результатів і час виконання** — показуються над таблицею.

### Записи

Управління всіма записами та їх редагування.

- **Форма «Додати запис»** — поля Id (можна залишити порожнім для автогенерації), Ключ (до 64 байт UTF-8) та Дані (до 128 байт UTF-8). Кнопка **«Додати»** створює запис; під формою відображаються помилки валідації.
- **Список усіх записів** — таблиця з пагінацією; внизу кнопки **◀ ▶** перемикають сторінки.
- **«Оновити»** — перезавантажує поточну сторінку зі сховища.
- **«Видалити вибрані»** — видаляє виділений у таблиці запис.
- **«Очистити всі»** — повністю очищує сховище після підтвердження.
- **Поле редагування** — з'являється під таблицею, коли вибрано рядок: дозволяє замінити поле «Дані» обраного запису кнопкою **«Оновити»**.

### Імпорт

Завантаження даних з CSV або генерація тестових наборів.

- **«Відкрити CSV»** — викликає системний діалог вибору файлу формату `Id;Key;Data`. Перші 10 рядків показуються у попередньому перегляді, далі застосунок одразу імпортує всі рядки файлу.
- **«Скасувати»** — з'являється під час тривалої операції; перериває її без втрати вже доданих записів.
- **Слайдер кількості та кнопка «Генерувати»** — створюють від 1 000 до 1 000 000 псевдовипадкових тестових записів.
- **Прогрес** — індикатор виконання та текст «Оброблено N з M записів».
- **Результат** — повідомлення з кількістю імпортованих та пропущених (дублікати, помилки) записів.

### Журнал

Хронологія останніх 100 операцій змін.

- **Випадаючий список фільтра** — «Всі операції» / «Вставка» / «Видалення» / «Оновлення».
- **Таблиця** — час, тип операції (кольорова мітка), Id запису, опис.
- **«Відмінити»** — скасовує останню операцію (вставку перетворює на видалення, видалення відновлює запис).
- **«Повторити»** — повертає скасовану операцію.
- **«Очистити журнал»** — стирає всю історію після підтвердження.

### Snapshot

Збереження та завантаження повного стану сховища у файл `.didx`.

- **«Зберегти Snapshot»** — викликає системний діалог збереження файлу.
- **«Завантажити Snapshot»** — повністю замінює поточний стан вмістом обраного файлу. Пошкоджені записи (з невалідним CRC-32) пропускаються, їх кількість виводиться окремо.
- **Інформаційна панель** — після успішної операції показує шлях до файлу, розмір, дату створення, кількість записів, версію формату та кількість пошкоджених записів.

### Аналіз продуктивності

Вбудований запуск тестів швидкодії на власному коді через `BenchmarkDotNet`.

**Передумова.** Перед першим запуском потрібно зібрати проєкт бенчмарків окремо:

```bash
dotnet build src/DIndex.Benchmarks -c Release
```

Це створить файл `src/DIndex.Benchmarks/bin/Release/net9.0/DIndex.Benchmarks.exe`, який застосунок знайде автоматично. Якщо файл відсутній, після натискання «Запустити аналіз» з'явиться червоне повідомлення з точною командою збірки.

> **Для запуску аналізу з `artifacts/DIndex.App.exe`** команду збірки бенчмарків треба виконати з кореня репозиторію (там де лежить `DIndex.slnx`). Застосунок автоматично знайде EXE за відносним шляхом `../src/DIndex.Benchmarks/bin/Release/net9.0/`.

- **Сценарії** (чекбокси) — обираються алгоритми для аналізу: BST vs лінійний пошук, сортування QuickSort, інтерполяційний пошук. Потрібно обрати хоча б один.
- **Розміри N** (чекбокси) — фільтр для таблиці результатів: визначає, які рядки показати після завершення. BenchmarkDotNet завжди запускає всі N, визначені у класі (`[Params]`), незалежно від цього вибору — тому на час виконання фільтр не впливає.
- **«Запустити аналіз»** — запускає `DIndex.Benchmarks.exe` у фоновому процесі. Виконання займає **кілька хвилин**; під час роботи відображається текстовий прогрес і лічильник часу.
- **«Скасувати»** — примусово зупиняє процес бенчмарку; з'являється лише під час виконання.
- **Таблиця результатів** — після завершення відображає: метод, N, середній час, стандартне відхилення, обсяг алокацій.
- **«Відкрити HTML-звіт»** — відкриває звіт у браузері: опис кожного сценарію, очікувана складність, таблиці результатів, короткі висновки і пояснення термінів. Звіт українською мовою.

## Встановлення та запуск

**Вимоги:** Windows 10 або новіше, .NET 9 SDK (для збірки з вихідного коду).

### Готовий застосунок

Запустити `artifacts/DIndex.App.exe` — файл самодостатній, встановлення .NET Runtime не потрібне.

### Запуск з вихідного коду

```bash
git clone <repo-url>
cd DIndex
dotnet run --project src/DIndex.App -c Release
```

### Публікація self-contained EXE

Щоб зібрати один самодостатній виконуваний файл з усіма залежностями всередині:

```bash
dotnet publish src/DIndex.App -c Release -r win-x64 --self-contained true \
  -p:PublishSingleFile=true \
  -p:IncludeNativeLibrariesForSelfExtract=true \
  -p:EnableCompressionInSingleFile=true \
  -o artifacts
```

Результат: `artifacts/DIndex.App.exe` (~61 МБ, .NET Runtime і всі нативні бібліотеки вбудовані всередині).

### Збірка модуля бенчмарків

Потрібна окремо — лише якщо планується використання сторінки «Аналіз продуктивності»:

```bash
dotnet build src/DIndex.Benchmarks -c Release
```

### Запуск тестів

```bash
dotnet test DIndex.slnx -c Release
```

Усього 41 тест; охоплюють BST, лінійні структури, сортування, пошук, PagedMemoryStore, транзакційний журнал і серіалізацію знімків.

### Локальне використання

Усі дані зберігаються лише в оперативній пам'яті поточного запуску. Для збереження між сеансами скористайтеся **Snapshot → Зберегти Snapshot**, файл `.didx` запишеться у вибрану вами папку. Наступного запуску відкрийте його через **Snapshot → Завантажити Snapshot**. Знімки і HTML-звіти аналізу залишаються на вашому диску; жодних мережевих з'єднань застосунок не виконує.
